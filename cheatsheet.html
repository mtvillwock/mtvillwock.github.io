Constructor Functions


          function objectName (optional_parameter) {
            var properties: {
              property: "thing",
              other: 23
            };
            function objectAction (optional_parameter) {
              this.method(); // objectName does something
            };
          }

          // Example

          function rookieCoder (name, age, education, language) {
            var properties: {
              name: name,
              age: age,
              education: "Dev Bootcamp",
              language: "Ruby"
            };
            function evaluateCode (string_of_code) {
              if (string_of_code.length > 20) {
                console.log("Man, that's a long method. Let's refactor.")
              }
              else {
                console.log("Succinct code. How refreshing.")
              }
            }
          }

Think of the constructor as the class, the properties (which don't need to be a variable, they can just be an object containing property : value pairs) as the initialize method, and the functions as methods (because functions inside functions are methods). Another useful resource to check out for this is here.

Using Bracket Notation to Add Properties

Let's say we want to make the code below, but on the fly. How can we do that?


          laptop = {
            make: "Apple"
            model: "2013 MacBook Retina Pro"
            programs: {
              browsers: ["Chrome", "Firefox", "Safari"]
              textEditors: ["Sublime Text 2", "Atom", "Brackets", "Text Wrangler", "Xcode"]
            };
          };

With dot notation and bracket notation! How do those work?

// Make an object literal
          laptop = {};
          // Make an object literal
          laptop.make = "Apple";
          // add the `make` property with a value of "Apple"
          laptop["model"] = "2013 MacBook Pro";
          // add the `model` property with a value  of "2013 MacBook Pro"
          laptop["programs"] = {};
          // add `programs` property with value of an object literal
          // the value of `programs` will become the objects (property: value pairs) that are added to it in the next two steps
          laptop["programs"]["browsers"] = ["Chrome", "Firefox", "Safari"];
          // add `browsers` property with value of an array of different browsers to `programs` object literal (which is technically also a property of `laptop`)
          laptop["programs"].textEditors = ["Sublime Text 2", "Atom", "Brackets", "Text Wrangler", "Xcode"];
          // add `textEditors` property with value of an array of different text editors to `programs` object literal (which is a property of `laptop`)

There are some differences between dot and bracket notation in terms of what you can access and how. Read more here. And remember: put those quotes around your properties! Like this:

object["property"] = "getting accessed"
