Choosing Your Data Plan: Hashes and Arrays

Oct. 25th, 2014

What are hashes and arrays?

Hashes are pairs of items sorted by keys and values.

Wait, what?

It's not as complicated as it sounds.

Imagine you are filling out a form. Name, address, city, and so forth are examples of keys. Correspondingly, the values would be the actual information that is being paired with the keys, for example: "Sherlock Holmes", "221B Baker St.", "London". This is similar to how symbols on a map indicate something else, such as a star for a capital city or a dot for a normal city.

What if you have data you want to store in a list instead of pairing things together?

Arrays are lists of objects that are indexed starting at zero.

Again, it's not as complicated as it sounds.

Arrays keep whatever is entered into them in the order it was received and assign an index number to each entry. For example, if you were to create an array with the entries "thing", "other thing", "different thing", those entries would be indexed at 0, 1, and 2 respectively. The way I remember this "start counting from zero" concept is to think of a zombie outbreak. Why? Because the first person infected is usually referred to as "Patient Zero".

How do I know which to use?

Hashes: Lists without Order

Hashes can be used for inventories, since different keys can have the same value. For example, in this hash,

store_inventory { 

              :eggs => 12,

              :bread => 3,

              :hamburgers => 4,

              :pizza => 3,

              }

there are equal amounts of pizza and bread.

Hashes are also useful for items that are logically paired together and lend themselves to the :key => "value" structure of a hash, but without the necessity of appearing in a linear numerical order. That could be the fields and answers of a form, or in the example below, players by position in a fantasy football roster.

my_roster {
              :qb1 => "Drew Brees",

              :wr1 => "Demaryius Thomas",

              :wr2 => "Roddy White",

              :wr3 => "Reuben Randle",

              :rb1 => "Shane Vereen",

              :rb2 => "Joique Bell",

              :te1 => "Rob Gronkowski",

              :flex => "Eddie Royal",

              :k => "Adam Vinatieri",

              :def => "San Francisco 49ers",

              }

Arrays: Step-by-Step Lists

Array are best used for lists, especially lists that need to be kept in order. This is because arrays associate their entries with an index number (starting with zero). For example, if you were listing finishers in a race, you might make an array like this:

runner_places = ["Frank", "Sam", "Tom"]

However, that would result in this when you printed it out:


            "Frank" => 0

            "Sam" => 1

            "Tom" => 2

Not exactly what we wanted, right? We can make this work if we feed an extra value to the array when we create it. Something like this:


            "Name/Place" => 0

            "Frank" => 1

            "Sam" => 2

            "Tom" => 3

Both hashes and arrays have their uses. You'll have to decide what is best to use for whatever your information storing needs.

What can I do with hashes and arrays after I make them?

Hashes and arrays aren't just storage containers for information. They can be updated, sorted, transformed, and manipulated by all sorts of methods (methods are the "verbs" as opposed to objects aka "nouns" in the Ruby programming language). However, since that isn't the focus of this post, I won't delve into that rabbit hole. Suffice to say, hashes and arrays are not limited to just holding onto data.
