Scoping Out Ruby Variables

Published Nov. 16th, 2014

Levels of Scope

Ruby supports five different kinds of variables with four levels of scope. Before we discuss the variables, let's cover scope to give some context.

The scope of a variable defines where the variable can be accessed from within a program. To use a (most likely imperfect) metaphor, you can think of different types of variables as having different security clearances. Ruby, much like in the government or some businesses, tends to work best using a need-to-know style of information access. This is both for security reasons and to reduce possible bugs or other unintended interactions.

So what are these different types of variables?

Local Variables

Ah, the humble local variable. Like its name suggests, local variables have a limited range. They only share their information within whatever method or loop that created them. Think of local variables as private conversations (no wiretaps here!). Information is shared where it needs to be, and only the people it pertains to have access to it.

secret_conversation = “The passcode is 1234.”

Local variables are useful for loops that are specific to a certain method, or when you want to make sure that the information contained in them is not accessible anywhere else. They must start with a lowercase letter or an underscore.

Class Variables

Class variables are accessible by any instance of a class. This can get dangerous, because if you change the variable value for one class, you'll change it for all instances of that class. However, if there is a class that you want to be able to adjust all instances of at once, class variables are one way to go.

@@classwide_conversation = “All officers must report to the barracks at 1600 hours.”
              # Wait, that’s not right. Better overwrite it.
              @@classwide_conversation = “All officers must report to the barracks at 0700 hours.”

Class variables are denoted with the @@ notation and must be initialized (have a value set) at creation.

Instance Variables

The little brother to class variables, instance variables can be accessed by one instance (see where that name comes from?) of a class. More precise than their class variable brethren, instance variables are great for when you have information within an instance of a class that you want to access at multiple points throughout that class.


              class Spy
              def initialize(location)
              @my_location = location
              end
              def call_for_extraction(hot_spot)
              @my_location = "14th and Richmond"
              puts "I need an extraction team ASAP at #{@my_location} now!"
              end
              def mission_complete(new_location)
              @my_location = "just outside target coordinates"
              puts "The job is done. I'm #{@my_location}, heading to the rendezvous."
              end
              end

Instance variables are also useful for when you want instances of a class to have an attribute that they can modify without it affecting other instances of the same class. Instance variables are notated with an @ symbol.

Global Variables

Global variables are variables that can be seen anywhere and changed from anywhere in the program. Using global variables is often frowned upon due to the combination of unnecessary security risk and debugging issues that they can create.

$nuclear_access_code = 1234567

As you can see from my hyperbolistic example, making certain variables global can have dire consequences. However, "a number of pre-defined global variables are available to obtain information about the Ruby environment." Global variables are written with a $ notation.

Constant Variables

Constant variables, despite what their name suggests, are not actually constant. They should be, but Ruby will allow you to change them after they are declared, but not without giving you a warning about doing so. The scope of a constant depends on where it is created. If the constant is made inside of a class, then it functions similarly to a class variable. If the constant is created outside of a class or module, it will have global scope.


              GLOBAL_INFO = { "secrets" => "Everyone know you can save money with Geico."}
              class InsuranceCompany
              CLASS_INFO = { "calculation" => "A times B times C equals X. If X is less than the cost of a recall, we don't do one."
              def initialize
              @insurance_rate = insurance_rate
              end
              end

Constants are written using a capital letter as the first letter (and sometimes in ALL_CAPS).

Closing the Scope

Based on my limited personal experience, I would say that instance variables are far and away my favorite variables to work with because they are accessible throughout an entire class. Local variables are helpful when iterating through loops, and if you need a once-of value in a pinch.

Class variables are cool, but can mess things up in a hurry, like if you changed the @@wheels variable on your class Automobile to 18 because you were making an instance of that class called semi_truck. Now all of your other instances of class Automobile would have 18 wheels. Global variables bring even more volatility to the table. Constants I could see using on occasion, but for now, I like to keep my information in the areas that I know it needs to be and nowhere else.

Hopefully this post left you feeling well-acquainted with Ruby's variables. Thanks for reading!
