AngularJS: The What and the Why

Published November 30th, 2014

What is AngularJS?

AngularJS is a JavaScript MVC framework for web applications. Two months ago I probably wouldn't have known what that meant, and it's possible I still don't fully understand it yet, but I'll try to explain as best I can.

Model-view-controller (MVC) is a method of structuring how a user interacts with software. Each of the three parts â€“ Model, View, and Controller have a different purpose. The model generally contains the data and rules/logic for how the application operates. This, as far as I understand it, includes the database and objects in the back-end code. The view is "what the user sees," which in the case of web application means whatever is in the browser window. The controller, as its name implies, controls actions that the view and the model take. This includes things like updating a database or scrolling up and down on a page. The controller is basically a middle man between the view and the model.


"MVC-Process" by RegisFrey - Own work. Licensed under Public domain via Wikimedia Commons.

Okay, so Angular is a JavaScript MVC. Cool. But what does it actually do? In this article from InfoWorld, Misko Hevery explains, "We take HTML, which is really good at static documents, and through this concept of directives, we add new markup to HTML that turns static content into dynamic content." Dynamic HTML... say what?

Yep, dynamic HTML. Basically what you do is use custom HTML tags to enable HTML pages to do things they were never designed to do. Is any of this revolutionary? Probably not. After all, you can do similar things with other methods (isn't that the story of most programming?). For example, jQuery is a popular library used for many front-end web app and/or design projects. So why use AngularJS?

Why is AngularJS useful?

Source: Implementing A Business Continuity Architecture by Christopher Janoch

The thing with code is that, like tribbles, it can multiply in problematic fashion. Sometimes such large amounts of code are necessary just to begin. I'm talking about boilerplate code, which is something that AngularJS cuts down on.

What wading through boilerplate code can feel like. Source: Memory Alpha

It's CRUD-y. CRUD, or "create, read, update, delete" is a common application style, used for everything from spreadsheets to booking plane tickets. "Angular was built with the CRUD application in mind". What this means to you is that unless you are building a game (in which case you could still probably use it), AngularJS is most likely built to handle what you're trying to make.

Other cool things about AngularJS is that it is "super-easily testable" and you are "freed from low-level DOM manipulation tasks," both of which sound pretty great to me.

I hope I've intrigued you with this discussion of AngularJS. Do you already have experience with other JavaScript frameworks? If so, what is your favorite and why?
