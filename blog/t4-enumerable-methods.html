Mapping Out Enumerable Methods

Enumer-What?

Enumerable. Enum. Enumerator. These words come up a lot in the Ruby documentation. But what the heck do they mean, and why should you care?

Here's my pitch.

Keep It Simple Concise

Enumerable is a Module in Ruby that is used in many different Classes, including arrays, hashes, and ranges.

"The Enumerable mixin provides collection classes with several traversal and searching methods, and with the ability to sort. The class must provide a method each, which yields successive members of the collection." ~ Ruby Documentation, Enumerable
Essentially it grants a Class (whether it is one you made, or a pre-built one), the ability to use variations of the #each method, such as #find, #group_by, and #map.

What's the catch?

The Class that includes the Enumerable module must "define an each instance method". (The Well-Grounded Rubyist, pg. 287).

What would that look like? 


                class NewClass
                  include Enumerable
                  def each
                    # "Behavior that yields items to a supplied code block one at a time" (pg. 287)
                  end
                end
Not a bad price to pay in order to access a bunch of useful data manipulation methods, right?

Mapping It Out

Okay, so what's an example of one of these much-hyped Enumerable methods?

How about #map?

The #map method has a few cool features. First, whatever information you feed to it will be returned as a new array. So if you had a range of numbers 1 through 10 and you wanted to square each of them, you'd do this: 


                range_mapped_to_array = (1..10).map { |x| x * x }
                #=> [1, 4, 9, 25, 36, 49, 64, 81, 100]

But what if you wanted to permanently make a change to your data?

That's where #map! comes in. What's the difference?

Pretty simple: #map is non-destructive and #map! is destructive. #map creates a copy of your data as an array, and then applies whatever transformations you feed to the code block (in the example above, 

|x|
represents each element in the array, and 
x * x
are the instructions for how to modify the array). #map! makes transformations to an existing array, permanently modifying it.

Here's an example:


              squared_array_destructively_mapped_to_floats = range_mapped_to_array.map! { |x| x.to_f }
              puts squared_array_destructively_mapped_to_floats
              #=>[1.0, 4.0, 9.0, 16.0, 25.0, 36.0, 49.0, 64.0, 81.0, 100.0]

Tada! Now you have an array of squared numbers that are floated (they have a decimal instead of being an integer). And they'll stay that way. I didn't have to assign a new variable name to the floated version of the array, because the original array that I created from mapping the range was destructively modified, but I did to illustrate what was happening.

However, thanks to my superfluous but descriptive variable, I now have two arrays with different names and the same transformed contents. Oops! Just something to remember. Be careful with 'destructive' methods, they have that descriptive adjective for a reason!

Note: Technically, #map! is an Array method, but using #map returns an array, so you can use #map to store something in an array and then permanently alter that array by using the #map! method.

I hope this explanation of the Enumerable module and the Enumberable#map method made sense.
