Never Finished

Bootstrap

Published December 7th, 2014

The Framework

Heading into today, I didn't much care for how my GitHub pages site looked. I had tinkered with it several times over the past week, but most of the changes weren't exciting and failed to make me feel good about the site. These week, I decided to try Bootstrap.

Bootstrap is a CSS framework created that allows faster, convenient creation of CSS stylesheets, which is incredibly useful for today's modern age of lightning fast startups. It has pre-made CSS ids and classes that are useful for common styling tasks, like making buttons or creating headers and footers.

For anyone interested in using Bootstrap, you can go here to download it. After you do that, walk through this course on making a website by Codecademy. They start using Bootstrap towards the last third of the course and it's a great introduction to the concepts.

How Far We've Come

It's been quite a journey. Less than nine weeks ago I had a web page that looked like something that a 12-year-old from the mid-90s had created. Now... well, I'll be honest. The giant header image with some nicely separated blocks of content underneath isn't exactly a new concept since Bootstrap emerged. But I suppose that's not Bootstrap's purpose. Bootstrap isn't intended to transform people into amazing designers, it's made to help developers get sites created.

Thank you, Bootstrap contributors. For someone like myself who is not the most visually inclined person, this CSS magic that you've conjured is amazing.

One of the most time consuming parts of maintaining this website over the past nine weeks was that every time I went to edit the CSS file, I'd spend at least an hour tweaking margins, padding, and colors in an effort to make it look nice. But with Bootstrap in my arsenal, I feel much more confidant in being able to create something that looks more like what I wireframed.

If nothing else, working on my own site has given me something to be a little proud of. Yes, it's relatively simple, and all I did was add two blog entries a week and tweak the CSS here and there. But it has been a growing experience. I've learned a lot about how plans shift over time and the importance of thinking things out before coding.

Thanks for joining me over the last nine weeks. I hope to see you around for the next nine!
